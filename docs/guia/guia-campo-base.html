¿Para quién es este módulo?

Para gente que se maneja con la terminal, sabe usar git y lee código sin que le sangren los ojos. No hace falta ser experto: hace falta no asustarse cuando algo falla. Si nunca has abierto una terminal, este no es tu punto de entrada — vuelve cuando lo sea, que el módulo no se va a ninguna parte.

BASE

Qué es CodeFlow y qué te llevas de aquí

CodeFlow es un proyecto de aprendizaje al estilo de la escuela 42 — nacido en 42 Urduliz, mi campus — para aprender a usar Claude Code como herramienta profesional. No «usar» de pedirle cosas y cruzar los dedos: usar de controlar. Configurarlo, vigilarlo, interceptarlo, integrarlo en tus propios programas.

El Módulo 00 tiene cinco ejercicios progresivos. Al terminarlos sabrás definir el contrato de trabajo de tu agente (CLAUDE.md), gestionar su memoria de trabajo, ponerle guardarraíles con hooks, conectarlo a herramientas externas via MCP, y llamarlo desde tu propio código C++ con el SDK. Eso, hoy, es una habilidad profesional con nombre y apellidos.

El trato

Este módulo es gratis, completo y sin trampa ni cartón. A cambio te pido una sola cosa: no te copies a ti mismo el aprendizaje. Si dejas que Claude (o esta guía) haga el módulo por ti, el único estafado eres tú.

MÉTODO

Aprender sin temario (y por qué funciona)

En 42 no hay clases, no hay apuntes, no hay profesor. Hay tres cosas: un subject (el enunciado del proyecto), tus compañeros, y tu capacidad de buscarte la vida. Suena a abandono y es justo lo contrario: es el sistema de aprendizaje más eficaz que he probado, y vengo de la carnicería y de la topografía antes que del código, así que de aprender de cero a los cuarenta y pico sé un rato.

¿Por qué funciona? Porque lo que encuentras tú, peleándolo, hace click y ya no se olvida. Lo que te dan masticado se evapora en dos semanas. La neurociencia lo llama aprendizaje activo; en 42 lo llamamos «búscate la vida». Mismo perro.

Si no tienes un campus lleno de compañeros a los que preguntar, esta guía hace de compañero de mesa: te orienta, te explica los conceptos y te avisa de dónde está el barro. Lo que no hace es resolverte los ejercicios — porque entonces dejaría de funcionar el método.

REGLAS

Las normas que descolocan al que viene de fuera

Cuatro reglas del subject que en 42 son el pan de cada día, pero que al que viene de cursos tradicionales le hacen cortocircuito. Mejor que te las cuente ahora y no que las descubras con un cero:

M/B

El bonus nunca compensa un mandatory incompleto. Cada ejercicio tiene parte obligatoria y parte bonus. Si la obligatoria no funciona completa, el bonus ni se mira. Traducción: nada de irte a hacer la parte chula dejando la base coja. Primero la cimentación, luego el ático.

N.md

Sin NOTES.md, el ejercicio vale 0. Aunque funcione. En cada ejercicio escribes, en tus propias palabras, qué hace tu solución y por qué funciona. No es papeleo: es el mecanismo anticopia más simple del mundo. Lo que no puedes explicar, no lo has aprendido — solo lo has ejecutado.

DEF

Todo se defiende. En 42, otro estudiante te evalúa cara a cara: te pide que expliques cada línea, que modifiques un hook en vivo, que ejecutes tu programa sobre un archivo que no has visto. Aquí no tendrás evaluador, así que cada capítulo de esta guía termina con un checkpoint: las preguntas que te haría yo. Hazte trampas ahí y el único que suspende eres tú.

GIT

Commit antes de soltar al agente. Siempre. Cada ejercicio en su rama, y checkpoint de git antes de dejar que Claude Code toque tu código. Trabajar con agentes sin red de seguridad no es de valientes, es de ingenuos.

EQUIPO

Lo que llevas en la mochila

Revisa esto antes de empezar — atascarse en el ex02 por no tener jq instalado es una muerte tonta:

  • Claude Code instalado y autenticado (cuenta Pro o superior, o API key de Anthropic Console)es la herramienta que vas a aprender a controlar — sin ella no hay módulo
  • Node.js ≥ 18lo necesitan el SDK (ex04) y los servidores MCP (ex03)
  • jq instaladopara inspeccionar el JSON de los hooks (ex02) — sudo apt install jq / brew install jq
  • Git, y soltura básica con ramas y commitscada ejercicio vive en su rama; el checkpoint previo es ley
  • Compilador C++ con -Wall -Wextra -Werror y -std=c++17los ejercicios con código compilan con flags estrictas, como debe ser
  • Tu editor de siemprevim, nvim, VSCode… aquí no hay guerra santa, usa el que te haga rápido
# Chequeo rápido de mochila
$ claude --version && node --version && jq --version && git --version
# Si las cuatro responden, estás listo. Si alguna falla, arréglala AHORA.
RUTA

El mapa del módulo

Cinco ejercicios, en orden de cota. Y el orden importa: cada uno construye sobre el anterior. El ex01 usa el proyecto del ex00, el ex04 recicla los hooks del ex02… Saltarte uno es subir una montaña empezando por la mitad.

Cómo combinar subject y guía: primero lee el subject del módulo entero — de verdad, entero, que el propio subject te lo pide y la gente no lo hace ni a tiros. Después, por cada ejercicio: lee el enunciado, lee el capítulo de esta guía, y al barro. El subject manda; la guía acompaña.
